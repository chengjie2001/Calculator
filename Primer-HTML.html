三、HTML 语义化

HTML的重点不在于掌握了多少标签（虽然标签确实很多，但凭借语义，能迅速记住它），因此，只要你能掌握语义的含义并且去尝试编写一个语义结构良好的页面，你的HTML会学的顺风顺水。

HTML的精髓就在于标签的语义。所以，请像写文章一样去写一个有结构层次感的网页

如同学习英语一般，几千个单词，用得越多，记得越多。对于HTML而言，常用的标签数量不到100个，凭借语义，是很容易记忆的。

什么是语义？

语义指的是：标签清楚地向浏览器和开发者描述其本身的意义。

h1标签向浏览器和开发者描述：这里有一个最重要的标题。

img标签向浏览器和开发者描述：这里有一个一张图片。

那么语义化有什么作用呢？

面向开发者：根据语义标签快速定位到对应位置的代码，从而利于开发调试和后期维护。像写文章的那样写一个页面，从代码结构和层次可以快速定位到代码。

面向浏览器：语义能够让搜索引擎小朋友不迷路，有利于搜索引擎小朋友更好的理解我们的网页，从而获取更多的有效信息，提升网页的权重。

HTML决定了网页的结构与内容，基础不牢，地动山摇，如何打好基础？就是凭借语义化标签

网页结构

内容不能杂乱无章，结构混乱，搜索引擎小朋友会迷路的

搜索引擎只会根据HTML代码来分析一个网站，识别到h1标签，知道这个是该网站的最高级标题了， 这就是这个网页最重要的信息，会做出相应的操作，遇到了p标签，知道这里有一段话了，遇到img标签，知道这里是一张图片。 如果整个页面都是div和span这样的无语义的标签，搜索引擎小朋友就会变得迷茫，就会离开这个让他不开心的地方，从而你的网站就崩了，谁也不会来看， 随之你也就崩了，最后你的BOSS也就崩了。

一句话就是：按照语义，将网页的结构梳理出来，然后利用特定的标签对将之按语义表示出来即可。关于如何使用语义标签，下章细讲。

在此废话那么多，不是告诉大家，语义化很重要，而是告诉大家，语义化是写网页的基本素质，“重要”二字也体现不出其价值。

你“块”乐吗？

大多数 HTML 元素被定义为块级元素（block）或内联元素(inline-block)。

内联元素在显示时通常不会以新行开始，定义 span，用来组合文档中的行内元素。

块级元素在浏览器显示时，通常会以新行来开始（和结束），通常定义文档中的分区或节。因此，块级元素+style，特别利于布局，因此div为布局而生。布局没有特殊含义（语义），仅仅是位置与颜值的改变，因此div没有语义！

所以当你写了如下代码：

<div style="font-size:36px;color: black;">一级标题</div> replace <h1>一级标题<h1>

会被很快乐的揍一顿，别问我为什么！好吧，既然你诚心诚意的发问了，我就勉为其难的告诉你吧。因为div没有语义，怎么能用来代表标题呢？ 打个比方来说：这样写代码，等于你写作文时用空白去表示一个标题，这听起来很滑稽，但事实就是如此滑稽，所以你会被快乐的揍一顿。

言归正传,div是块状元素，span是内联元素，二者都无语义

需要注意的是：

  1. 一个页面只能有一个h1，h1会被赋予最高权重。像写文章一般，我们写作文不会写多个主标题，因此一个页面只能写一个h1。
  2. h1和h6之间不要出现断层，这样确保层次不断，由最重要到局部内容。
  3. 我们可以给h1~h6定义样式，但我们不能用h1~h6来定义样式

    4. 很多时候我们要为文本定义大小，有些人喜欢用hn标签来替代写style属性，使用标签控制样式，这样做， 你会被快乐的捶一顿的。HTML关注语义，通过语义分析结构，style关注样式，而这关注点不同，因此结构与样式不可混为一谈，结构和样式应当分离。但我们可以给h1加上样式。

  4. 千万不要用div来代替标题，div没有语义！用div代替h1~h6，是一个大通病。

一个表格被分为三部分 thead,tbody,tfoot（表头，表身，表脚）三部分。tfoot用的很少，但它体现的是表格结构的完整。

表格的三部分，每一个部分都以“行”为基准，“单元格”会分解“行”，因此 tr(table-row）必须嵌套 th,td(table-cell),这样能体现出结构的完整。

HTML中零散的语义标签

换行符 <br/>

<br> 可插入一个简单的换行符。

<br> 是空标签，它没有结束标签，因此推荐大家这样写：<br/>，把结束标签放在开始标签中

很多萌新会使用<br/>来进行结构换行，

<br>这是错误写法，br没有结束标签</br> <span>1<span> <br/> <span>2<span> <br/>
这是错误写法，br没有结束标签
1
2

以上示例是错误的用法

<br/>有自己特定的语义：对文本换行，不能用于别的情况。

图片 <img/>

同理：img要这样写：

<img src=".../.../..." alt="给浏览器看的图片描述" title="给用户看的图片描述" />

img标签要正确的结束需要在最后加一个“/”

无序列表 <ul/>

列表类型的数据，推荐采用ul+li标签

不推荐使用div+div这样的无语义标签,因为不仅丧失了语义，还因为布局常用div布局，如果div标签过多，不利于维护

<div> <div>1.1</div> <div>2.1</div> <div>3.1</div> </div>
1.1
2.1
3.1

因为有序列表的样式固定，所以大多数采用无序列表+span自定义格式去实现效果，关于style的一切，都在CSS教程里面，此处可以跳过看不懂的style，看看就好。

常见的块级元素：

  • 1div（用于布局）
  • 2h1~h6（用于标题）
  • 3table（用于表格展示数据）
  • 4ul（呈现无序列表）
  • 5ol（呈现有序列表）
  • 6p（用于写一段话）

常见的内联元素：

  • 1span（用于组合文档中的行内元素）
  • 2a（用于链接）
  • 3strong（用于强调：加粗）
  • 4em（用于强调：斜体）
  • 5img（用于展示图片）

强调字符的标签<strong><em>

二者均为内联标签，均会增强文本的权重

样式表现为:<strong>:加粗文本,<em>：文本倾斜。

<p>样式表现为:<strong>&lt;strong&gt;:加粗文本。</strong> <em>&lt;em&gt;：斜体</em></p>

样式表现为:<strong>:加粗文本， <em>：文本倾斜。
